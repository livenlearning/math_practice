1. Edit the Rules

Algebra → Game

These rules are written in code, but they’re just algebra expressions in terms of level. Change the expressions, click Apply rules & restart, then play the game on the right.

Enemy speed rule (per enemy) speed(level)

Example: 2 + 0.3 * level means “start at 2, then add 0.3 times the level.”

Spawn interval rule (frames between enemies) interval(level)

Example: 120 - 5 * level means “start at 120 frames, then subtract 5 each level.” Smaller interval = more enemies.

Score gain rule (per frame) scoreStep(level)

Example: 1 * level means “gain more points on higher levels.”

Edit the expressions, then click Apply rules & restart. If there’s an error, you’ll see a message here.

You can use +, -, *, / and numbers, with the variable level. The game will test your rule for several levels (like level 1, 3, and 5) before using it.

Use the left and right arrow keys to move the blue player. Avoid the red squares. Survive longer for more points.

2. How to Read These Rules

Connect to Algebra

Each rule is a function of level. For example:

  • speed(level) = 2 + 0.3 * level → Faster enemies on higher levels.
  • interval(level) = 120 - 5 * level → Enemies appear more often on higher levels.
  • scoreStep(level) = 1 * level → You earn more points each frame on higher levels.

Try changing the numbers and then tell Dad (or write down) what the new rule is in words, like: “Enemy speed equals 1 plus 0.5 times the level.”
